CURSOR PROMPT-PACKдля жёсткой доработки UI/UX слоясистемы «СкладПро»
Документ для прямой загрузки в Cursor AI
Назначение: дать Cursor AI не абстрактные пожелания, а конкретные инструкции по переработке интерфейса. Все prompts ниже написаны так, чтобы Cursor делал реальные экраны, компоненты, состояния, таблицы, модальные окна, боковые панели и mobile-сценарии без самодеятельности.
1. Как использовать этот документ
Сначала вставь в Cursor «Мастер-промпт проекта».
Затем работай по модульным промптам по одному блоку: Dashboard → Склады → Поступления → Выдачи → Перемещения → Материалы → Наряды → Заявки → Лимиты → Согласования → Документы → Администрирование.
После каждого модуля проси Cursor показать: структуру компонентов, роуты, ключевые состояния, edge cases и готовый код.
Запрещай Cursor менять бизнес-логику без явного разрешения: он должен улучшать UI/UX, а не перепридумывать процессы.
2. Мастер-промпт проекта для Cursor
Скопируй целиком следующий prompt:
Ты работаешь над жёсткой переработкой UI/UX слоя складской системы «СкладПро» для строительной компании.Твоя задача: не косметически «причесать» интерфейс, а полностью перевести текущий грубый админский UI в современную, плотную, понятную, производственную систему уровня enterprise-light.Требования:1) Стек UI: React + TypeScript + Tailwind + shadcn/ui.2) Нельзя ломать существующую бизнес-логику складских операций.3) Нужно улучшить:- информационную архитектуру,- навигацию,- таблицы,- карточки сущностей,- формы прихода/выдачи/перемещения,- систему статусов,- мобильные сценарии,- QR-поток для инструмента,- документы и файлы,- согласования и контроль лимитов.4) Новый UI должен быть визуально близок к следующему направлению:- тёмный левый сайдбар,- светлый рабочий canvas,- плотные карточки KPI,- современные таблицы с фильтрами и сохранёнными представлениями,- правые drawer-панели для деталей,- цветные статусы,- быстрые действия,- sticky header,- адаптация под desktop + mobile + PWA.5) Все действия пользователя должны быть быстрее текущих минимум в 2 раза по количеству кликов.6) Для каждой страницы нужно продумать состояния:- empty,- loading,- success,- validation error,- permission denied,- no data,- partial data,- conflict / warning.7) Для всех основных таблиц нужны:- поиск,- фильтры,- сортировка,- сохранённые фильтры,- массовые действия,- экспорт,- быстрый просмотр строки без перехода на новый экран.8) Для форм нужны:- секции,- inline validation,- keyboard-first UX,- горячие клавиши,- sticky footer с действиями,- режим черновика.9) Выдача ТМЦ, заявки и лимиты — критические потоки. Их нужно проработать глубже остальных.10) Не делай generic admin panel. Делай интерфейс именно для складской системы строительной компании.Выводи результат в таком формате:- карта страницы,- список компонентов,- UX-логика,- states,- data model на уровне props/interfaces,- код экрана,- код подкомпонентов,- примечания по mobile.
3. Пакет модульных промптов
Ниже — готовые промпты для Cursor по каждому экрану. Запускай их последовательно.
3.1 Dashboard / Главная
Переработай экран Dashboard.Нужно сделать:- верхнюю sticky-панель с глобальным поиском, переключателем объекта, уведомлениями и профилем;- KPI-карточки: поступления, выдачи, перемещения, лимиты, на согласовании, просроченные задачи;- блок “остатки по складам” в виде donut + breakdown;- таблицу “топ материалов по остатку / резерву / доступному количеству”;- блок “последние поступления” со статусами;- блок “открытые выдачи на бригадиров”;- блок быстрых действий: новое поступление, новая выдача, перемещение, наряд, заявка;- правую колонку attention panel: просрочки, согласования, проблемы лимитов.Нужно продумать:- drilldown из каждой карточки;- desktop grid;- compact mobile dashboard;- skeleton loading.Сделай полноценный production-ready экран и набор компонентов.
3.2 Склады и остатки
Переработай экран “Склады и остатки”.Нужно:- карточки складов сверху с адресом, ответственным и быстрым входом;- табличный блок остатков с поиском, фильтрами по складу/категории/нулевым остаткам/резерву;- статусы: в норме, мало, критично, зарезервировано;- row quick preview в правом drawer;- history tab: приход / выдача / перемещение по выбранному материалу;- массовые действия: экспорт, печать, резервирование;- saved views.Сделай интерфейс не как пустую форму, а как рабочую складскую таблицу.
3.3 Поступления
Полностью переработай UX экрана “Поступления”.Сценарий:- выбор склада,- тип поступления,- поставщик,- номер документа,- дата,- добавление строк материалов,- загрузка УПД/ТН/фото,- проверка совпадений номенклатуры,- черновик / провести.Нужно сделать:- секционную форму;- умное добавление строк;- поиск материала в строке;- быстрый ввод количества;- sticky action bar;- вкладки: Основное / Материалы / Файлы / История;- справа summary card.Добавь состояния:- дубль документа,- материал не найден,- частично заполнено,- ошибки валидации,- конфликт по номенклатуре.
3.4 Выдача ТМЦ
Сделай главный UX-поток системы: “Выдача ТМЦ”.Требования:- шапка выдачи с объектом, складом, бригадиром, прорабом, проектом, основанием;- обязательная проверка лимита и остатков;- табличный блок материалов и инструмента;- визуальные индикаторы: лимит / выдано / остаток / превышение;- right drawer подробностей выдачи;- вкладки: Основное / Материалы / Инструмент / Файлы / История;- статусы выдачи;- кнопки: сохранить черновик, провести, отправить на согласование, возврат, печать.Добавь UX для:- частичной выдачи,- превышения лимита,- срочной выдачи,- выдачи инструмента по QR.Сделай экран флагманским, намного сильнее типичной админки.
3.5 Перемещения
Переработай экран “Перемещения”.Нужно:- from warehouse / to warehouse;- проверка доступности на складе-отправителе;- транспортная накладная;- ответственные;- статусы: draft / approved / shipped / received;- timeline этапов;- список позиций;- actions footer.Добавь отдельный список всех перемещений с фильтрами и цветными статусами.
3.6 Материалы / номенклатура
Сделай современный экран номенклатуры.Нужно:- master-detail layout;- слева таблица материалов;- справа карточка материала;- табы: Основное / Приходы / Выдачи / Перемещения / Синонимы / Документы;- быстрый поиск;- фильтры по категории, единице, активности;- флаг дублей;- блок синонимов и альтернативных названий;- действия: объединить, архивировать, экспортировать.Нужен сильный data-heavy UI.
3.7 Наряды на работы
Сделай переработку экрана “Наряды”.Нужно:- split layout: список нарядов + форма/карточка;- поля: номер, тема, проект, объект, склад отгрузки, бригада, период;- блок перечня работ;- блок материалов в заявке кладовщику;- статусы наряда;- кнопки: сохранить, отправить кладовщику, PDF.Сделай экран похожим на production work-order tool.
3.8 Заявки на выдачу
Сделай экран списка и карточки заявок.Нужно:- список заявок с колонками: статус, склад, проект, наряд, бригадир, дата, сумма позиций;- фильтр по “на согласовании”, “к выдаче”, “отклонено”, “в работе”;- drawer с деталями заявки;- переход в выдачу;- visual status chips.Добавь mobile сценарий подтверждения заявки.
3.9 Лимиты по проектам
Переработай экран лимитов.Нужно:- выбор проекта;- сводная шапка по лимитам;- таблица материалов с колонками: план, выдано, резерв, остаток, перерасход;- визуальные progress bars;- комментарии и согласованные замены;- история версий лимитов;- footer editor только для разрешённых ролей.Сделай так, чтобы лимиты воспринимались как контрольный центр, а не пустая форма.
3.10 Согласования
Сделай нормальный экран согласований.Нужно:- отдельные типы согласований: перерасход, замена материала, срочная выдача, безлимитная выдача;- список карточек / таблица;- severity / priority;- side panel с контекстом;- approve / reject / request changes;- комментарии;- история решений.Нужен интерфейс, в котором руководителю удобно принимать решения за 30 секунд.
3.11 Документы + QR
Переработай экран документов.Нужно:- tabbed document center: УПД / ТН / фото / акты / прочее;- drag&drop upload;- preview panel;- фильтры;- entity links;- история файлов;- генератор QR для инструмента в правой панели;- карточка инструмента рядом с QR.Сделай файловый центр современным и удобным.
3.12 Пользователи / аудит / безопасность
Сделай три связанных экрана:1) Пользователи и роли2) Журнал аудита3) Профиль и безопасностьНужно:- users table + details panel;- role badges;- granular permissions view;- audit log with filters and chips;- security settings: 2FA, Telegram, email, active sessions;- alert banners по рискам.Это должен быть не унылый админ-блок, а аккуратная control panel.
4. Жёсткие UI-правила для Cursor
Запрещено делать пустые серые экраны с одной таблицей и без иерархии.
Запрещено растягивать форму на всю ширину без секций и смысловых контейнеров.
Запрещено делать destructive action primary-кнопкой.
Запрещено использовать только модальные окна там, где уместнее drawer.
Каждый критический экран должен иметь: header, primary action, secondary actions, filters, content area, contextual details.
Все числа остатков и лимитов должны быть визуально сравнимы: текст + цвет + progress/indicator.
Таблицы должны поддерживать dense mode и comfortable mode.
Все статусы должны иметь единый color/token mapping.
Все формы должны работать с клавиатуры.
Mobile не должен быть просто ужатым desktop — нужны отдельные сценарии.

6. Design tokens и визуальное направление
Primary: глубокий синий.
Success: зелёный для подтверждённых и проведённых статусов.
Warning: янтарный для ожидания и частичных кейсов.
Danger: красный для превышений, споров и просрочек.
Surface: светлый фон canvas + белые карточки.
Sidebar: тёмный navy.
Радиусы: 12–16 px на карточках, 10–12 px на полях.
Тени: мягкие, не тяжёлые.
Типографика: Inter / system sans, плотная, читаемая.
Иконки: lucide-react.
7. Prompt для mobile/PWA слоя
Сделай mobile-first слой для СкладПро как PWA.Нужны экраны:- мобильная главная,- список выдач,- карточка выдачи,- сканирование QR,- материалы,- задачи/согласования.Требования:- bottom navigation,- крупные tappable зоны,- быстрые сценарии в 1–2 шага,- camera/QR flow,- offline-friendly states,- compact KPI cards,- no desktop leftovers.Сгенерируй mobile components и экраны отдельно от desktop.
8. Definition of Done для UI-переработки
Есть единый AppShell и дизайн-система.
Все ключевые экраны переведены на новую структуру.
Критические потоки: поступление, выдача, перемещение, лимиты, согласования — реализованы с новыми UX-сценариями.
Таблицы поддерживают фильтры, сортировку, быстрый просмотр, экспорт.
Документы и QR-контур встроены в интерфейс.
Есть mobile/PWA-экраны для полевых сценариев.
Нет ни одного экрана, выглядящего как старая пустая админка.
Код разбит на переиспользуемые компоненты.

Покажи types/interfaces.

Покажи states и edge cases.
Покажи production-ready code файла страницы и дочерних компонентов.
Покажи, что изменилось по UX относительно старого экрана.
